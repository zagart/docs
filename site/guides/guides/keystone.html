Keystone — Setup Guide

net.zagart.keystone:core:X.Y.Z

Published via: GitHub Packages · Authentication required (PAT with read:packages)

Installation

1. Authentication

Create a GitHub Personal Access Token with read:packages scope, then add to ~/.gradle/gradle.properties:

gpr.user=YOUR_GITHUB_USERNAME
gpr.token=YOUR_GITHUB_PERSONAL_ACCESS_TOKEN

For CI, use environment variables:

# In your CI config:
GITHUB_ACTOR=${{ github.actor }}
ACCESS_TOKEN=${{ secrets.ACCESS_TOKEN }}

2. Add GitHub Packages repository

In settings.gradle.kts:

dependencyResolutionManagement {
    repositories {
        google()
        mavenCentral()
        maven {
            name = "GitHubPackages"
            url = uri("https://maven.pkg.github.com/zagart/keystone")
            credentials {
                username = System.getenv("GITHUB_ACTOR")
                password = System.getenv("ACCESS_TOKEN")
            }
        }
    }
}

3. Add dependency

commonMain.dependencies {
    implementation("net.zagart.keystone:core:X.Y.Z")
}

Quick Start

val storage = Keystone.Storage.create(name = "my_app")

// Write
storage.putString("user", "name", "Alice")
storage.putBoolean("user", "notifications", true)

// Read
val name = storage.getString("user", "name", default = "Guest")

// Observe
storage.observe("user") {
    string("name")
    boolean("notifications")
}.collect { snapshot ->
    updateUI(snapshot)
}

Entity Persistence

@Entity(type = "user_profile")
data class UserProfile(
    @Key val id: String,
    val displayName: String,
    @Embedded(prefix = "prefs_")
    val preferences: UserPreferences
)

// Save & load
storage.putEntity(profile)
val loaded = storage.getEntity<UserProfile>("user_123")

Publishing

To publish a new version:

  1. Update version in core/build.gradle.kts
  2. Run ./gradlew :core:publishAllPublicationsToGitHubPackagesRepository
  3. Tag and push: git tag vX.Y.Z && git push origin vX.Y.Z

Requires gpr.user / gpr.token in ~/.gradle/gradle.properties with write:packages scope.
