Quick Start

1. Theme Setup

Wrap your app content in DesignTheme. It provides LocalScreenConfiguration and applies the selected color scheme, typography, and shapes.

import net.zagart.design.ui.DesignTheme
import net.zagart.design.ui.Settings

val settings = Settings(
    isDark = true,
    animateTransitions = true,
    theme = Theme.Indigo
)

DesignTheme(settings = settings) {
    // content
}

Three theme variants are available:

  • Indigo — blue-indigo palette
  • CyanDeep — deep cyan-teal palette
  • OliveGold — warm olive-gold palette

2. Basic UI Components

import net.zagart.design.ui.components.*
import net.zagart.design.ui.components.button.Button
import net.zagart.design.ui.components.listitem.ListItem

        ListItem(label = "Notifications", onClick = { /* navigate */ })
        SwitchListItem(label = "Dark mode", checked = isDark, onCheckedChange = { isDark = it })
        Button("Save", onClick = { /* save */ })
    }
}

Builder DSL — Screen-Driven UI

Architecture

The Builder module separates UI rendering from state management via the ScreenController interface. The controller owns the screen state as a StateFlow, while ScreenComposable observes it and renders the appropriate ScreenState variant.

Creating a ScreenController

import net.zagart.design.builder.*
import net.zagart.design.builder.models.*

class MyScreenController : ScreenController {

    private val _state = MutableStateFlow(buildInitialState())
    override val state: StateFlow<ScreenState> = _state
    override val isNavigationBarVisible = MutableStateFlow(true)

    private fun buildInitialState(): ScreenState = screen {
        header {
            title("My Screen")
            search()
        }
        section {
            item(id = "opt_a", label = "Option A", subtitle = "Description A")
            toggle(id = "feature_x", label = "Enable Feature X")
        }
        lane {
            media(id = "m1", title = "Item 1", imageUrl = "https://...")
            media(id = "m2", title = "Item 2", imageUrl = "https://...")
        }
        choice {
            option(id = "sort_date", label = "Sort by date")
            option(id = "sort_name", label = "Sort by name")
        }
        filter {
            option(id = "cat_a", label = "Category A")
            option(id = "cat_b", label = "Category B")
        }
    }

    override fun onClick(id: String) {
        // handle component click by id
    }

    override fun onSearch(query: String) {
        // update state with filtered results
    }

    override fun onRefresh() {
        _state.value = loading { message("Refreshing...") }
        // reload data, then emit new screen { ... }
    }

    override fun onBack() { /* navigate back */ }
    override fun onLoadMore() { /* paginate */ }
    override fun onResume() { /* refresh on return */ }
    override fun isChecked(id: String) = false
    override fun isSelected(id: String) = false
    override fun isFiltered(id: String) = false
    override fun filter(filterId: String, optionId: String) { }
    override fun map(itemId: String) { }
    override fun registerLazyLoader(componentId: String, loader: () -> Unit) { }
}

Rendering

import net.zagart.design.builder.ScreenComposable
import net.zagart.design.ui.DesignTheme

DesignTheme(settings) {
    ScreenComposable(controller = myController)
}

For stateless usage with explicit callbacks:

import net.zagart.design.builder.ScreenComposableUi

ScreenComposableUi(
    state = myController.state.value,
    onRefresh = myController::onRefresh,
    onLoadMore = myController::onLoadMore,
    onClick = myController::onClick,
    onBack = myController::onBack,
    onSearch = myController::onSearch,
)

Loading & Error States

// Loading state with optional message
_state.value = loading { message("Loading content...") }

// Error state with optional retry
_state.value = error {
    message("Something went wrong")
}

Architecture Overview

The library follows a data-driven UI pattern:

  1. UI module — stateless, themed Composables (buttons, lists, navigation, inputs)
  2. Builder module — declarative DSL producing ScreenComponentViewData models, dispatched via ComponentMapper to UI renderers
  3. ScreenController — decouples state from rendering, enabling both local and backend-driven UIs

This architecture enables rendering any screen state uniformly, whether built programmatically or parsed from a server response.

Versioning & Releases

  1. Update version in ui/build.gradle.kts and builder/build.gradle.kts
  2. Run ./gradlew :ui:publishAllPublicationsToLocalRepository :builder:publishAllPublicationsToLocalRepository
  3. Tag and push: git tag v1.0.0 && git push origin v1.0.0
  4. Maven Central deployment happens automatically via daily CI
